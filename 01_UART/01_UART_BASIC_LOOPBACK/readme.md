## Design Theory

### UART Communication

UART (Universal Asynchronous Receiver Transmitter) is a serial communication protocol that transfers data one bit at a time.

A UART frame consists of:

```text
Start Bit | 8 Data Bits | Stop Bit
     0    |   D0-D7     |    1
```

The transmitter converts parallel data into serial form, while the receiver converts the serial data back into parallel form.

---

### Architecture Used

The design consists of:

```text
                +---------------------+
                | Baud Rate Generator |
                +---------------------+
                         |
                 ----------------
                 |              |
                 v              v
          +-------------+  +-------------+
          | Transmitter |  |  Receiver   |
          +-------------+  +-------------+
                 |              ^
                 |              |
                 +------TX------+
```

The transmitter output (`tx`) is directly connected to the receiver input (`rx`) inside the top module. This configuration is called **loopback communication**.

---

### Shared Baud Tick

In this implementation, both transmitter and receiver use the same baud enable pulse generated by the baud rate generator.

```text
Baud Generator
      |
      +---- tx_enb ---> Transmitter
      |
      +---- rx_enb ---> Receiver
```

Since both modules operate using the same timing reference, the receiver samples data exactly when the transmitter changes bits.

This simplifies the design and makes it suitable for:

- Learning UART fundamentals
- FSM design practice
- RTL verification
- Loopback testing

---

### Limitation of This Approach

Although this design correctly demonstrates UART communication, it is a simplified implementation.

In a practical UART system:

- The transmitter and receiver do not share a timing signal.
- Only the serial data line is shared.
- The receiver must recover timing from the incoming data stream.

Therefore, industrial UART receivers typically use:

- Start-bit detection
- Mid-bit sampling
- 16× oversampling

to tolerate clock differences between transmitter and receiver.

---

### Baud Mismatch Experiment

During development, different baud rates were intentionally assigned to the transmitter and receiver.

Observation:

```text
TX Baud = RX Baud
→ Correct reception

Small mismatch
→ Bit errors

Large mismatch
→ Corrupted data
```

This experiment demonstrated the importance of timing synchronization in UART communication and motivated the use of oversampling techniques in practical UART receivers.
